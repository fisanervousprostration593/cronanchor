# Design System: CronAnchor — MASTER

> Canonical, binding source of truth for the build. Generated by the design-intelligence
> engine (Step 2) and enriched with `--domain` style/color/typography/ux deep-dives and
> the `--stack` guidelines (Step 3–4). Page-specific files in `pages/` override this
> where they differ; otherwise MASTER governs. Every value below is a contract, not a
> suggestion — any drift in the implemented UI is a defect.

## Pattern

- **Name:** Tool-First Single Column (adapted from "Minimal Single Column")
- **Focus:** One primary job — configure a cadence and get a correct, copyable result.
  The "CTA" is the generated output + copy actions, not a marketing button.
- **Layout:** Single centered column, generous whitespace, no nav clutter, mobile-first.
- **Section order (top → bottom):**
  1. **Header** — product name + one-line value prop ("verified before you install").
  2. **The problem** — compact "why plain cron is wrong" explainer (the `*/14` trap).
  3. **Configurator** — cadence mode, interval, weekday, time, anchor date, timezone.
  4. **Output** — cron line · shell guard · full crontab block, each with a copy button.
  5. **Next-20 preview** — the real fire dates, the proof the schedule is right.
  6. **Footer** — offline/privacy note, GNU-coreutils note, license link.

## Style

- **Name:** Dark Mode (OLED) — refined "developer console"
- **Mode Support:** **Dark only** (light mode is an explicit anti-pattern here).
  Set `color-scheme: dark`.
- **Keywords:** dark theme, high contrast, deep slate, eye-friendly, technical,
  precise, terminal-adjacent, calm (not neon-cyberpunk).
- **Performance:** ⚡ Excellent · **Accessibility target:** WCAG AAA where feasible
  (text contrast ≥ 7:1 on background; never below 4.5:1).
- **Note:** Lean on a slate background (`#0F172A`), not pure `#000000` — easier on the
  eyes for a read-heavy utility while keeping OLED-friendly darkness.

**Refinements (intentional, product-voice):**
- `--color-ring` is set to a bright sky `#38BDF8` (not the engine's dark `#1E293B`) so
  keyboard focus is unmistakably visible against dark panels.
- `--color-warning` `#F59E0B` (amber) added for the "why cron is wrong" callout and to
  mark the broken `*/14` example — semantic, not decorative.
- `--color-destructive-text` `#F87171` is the lighter red used for error **text** on dark
  surfaces (≥ 5:1 contrast on card and on the destructive tint, clearing the 4.5:1
  floor). `#EF4444` stays for error **borders/accents** (a 3:1 non-text rule), where its
  contrast is fine.
- Accent green `#22C55E` is reserved for correctness signals (valid output, "copied",
  the fire-date markers). Never use it for plain decoration.

## Typography

Pairing = **"Developer Mono"** from the typography deep-dive (overrides the engine's
Inter-only default — the product's primary content *is* code, so a real mono matters).

- **Headings / code / cron / dates:** **JetBrains Mono** (`--font-mono`)
- **Body / labels / UI:** **IBM Plex Sans** (`--font-sans`)
- **Mood:** code, developer, technical, precise, functional.
- **Google Fonts CSS import (source for vendoring):**
  ```css
  @import url('https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@300;400;500;600;700&family=JetBrains+Mono:wght@400;500;600;700&display=swap');
  ```
- **Type scale (rem):** 0.78 (caption) · 0.875 (small) · 1 (body) · 1.125 (lead) ·
  1.5 (h3) · 2 (h2) · 2.5 (h1, `clamp(2rem, 5vw, 2.5rem)` responsive).
- **Rules:** all cron/shell/date output is JetBrains Mono. Body line-height ≥ 1.5.
  Numbers in the preview use `font-variant-numeric: tabular-nums` so columns align.
- **Self-host the fonts** (vendored into the build) so the runtime makes **zero network
  calls** — the Google Fonts URL above is the source, not a runtime `<link>`.

## Spacing & radii

- **Spacing scale (px):** 4, 8, 12, 16, 24, 32, 48, 64 → `--space-1 … --space-8`.
- **Radii:** `--radius-sm` 6px (inputs, chips), `--radius-md` 10px (cards/panels),
  `--radius-lg` 14px (the main container). Consistent, never mixed ad hoc.
- **Container:** max-width ~720px, centered; responsive padding (see stack guidelines).

## Key Effects

- Subtle only. Transitions **150–300ms** ease on hover/focus/state changes.
- Minimal accent glow (`box-shadow`/`text-shadow: 0 0 10px` at low alpha) used
  **sparingly** on the active/correct state — never everywhere.
- Visible **focus-visible** ring (2px, `--color-ring`) on every interactive element.
- Copy feedback: brief accent-green confirmation ("Copied"), then revert.
- **No** continuous/decorative animation. Motion only conveys state (loading, copied).

## Avoid (Anti-patterns)

Merged from the engine baseline + the UX deep-dive. Each is a defect if it appears:

- ❌ Light mode default / any white (`#FFFFFF`) background surface.
- ❌ Emojis used as functional icons — use inline **SVG** (Heroicons/Lucide-style).
- ❌ Continuous/decorative animation (e.g. bouncing icons); infinite motion that isn't
  a loading indicator. *(UX: "Continuous Animation", severity Medium)*
- ❌ Frozen UI with no feedback during async/compute work — show state.
  *(UX: "Loading States", severity High)*
- ❌ `z-index: 9999` hacks; fighting the stacking context instead of isolating it.
  *(UX: "Stacking Context", severity Medium)*
- ❌ `:focus` rings that show on mouse click — use `:focus-visible`.
- ❌ Low-contrast grey-on-grey text (must clear 4.5:1; aim 7:1).
- ❌ Neon-cyberpunk overload (glitch/scanlines) — rejected for accessibility/calm.
- ❌ Slow first render / layout shift.

## Stack guidelines (html / css)

From the `--stack html-tailwind` deep-dive, translated to the vanilla-CSS build:

- **Responsive padding:** scale container padding by breakpoint — e.g.
  `padding-inline: clamp(16px, 4vw, 32px)` (don't use one fixed padding at all sizes).
- **Reduced motion (High):** wrap all transitions/animations so
  `@media (prefers-reduced-motion: reduce)` disables them.
- **Focus visible (Medium):** style `:focus-visible` (keyboard), not `:focus` (also
  fires on click), with a 2px `--color-ring` outline/box-shadow.

## Pre-Delivery Checklist

Run this before calling the UI done (re-checked in Phase 5):

- [ ] No emojis as icons (inline SVG only).
- [ ] `cursor: pointer` on all clickable elements.
- [ ] Hover states with smooth transitions (150–300ms).
- [ ] Text contrast ≥ 4.5:1 everywhere (target 7:1 on background).
- [ ] Focus states visible for keyboard nav (`:focus-visible`, `--color-ring`).
- [ ] `prefers-reduced-motion` respected (motion disabled when set).
- [ ] Loading/compute + empty + error + success states all present and styled.
- [ ] Responsive at 375 / 768 / 1024 / 1440 px (no overflow, no layout break).
- [ ] Dark-only; `color-scheme: dark`; no white backgrounds.
- [ ] Monospace for all cron/shell/date output; `tabular-nums` for date columns.
- [ ] Zero runtime network calls (fonts self-hosted/vendored).
